Motivation::
The simplest possible deployment: one container, one file, one command.
`docker compose up -d` starts the application.
`docker compose down` stops it without touching the data volume.

Quality and Performance Features::
* The named volume survives container recreation and image upgrades.
* The multi-stage Docker build produces a minimal image (scratch or distroless base)
  with only the compiled Rust binary and the WASM assets.
* TLS termination, certificate management, and HTTP-to-HTTPS redirection are
  delegated to the operator's existing reverse proxy.

Mapping of Building Blocks to Infrastructure::

[cols="1,2", options="header"]
|===
|Building Block |Infrastructure Element

|Axum Router + Auth Middleware
|`appProcess` (Rust binary, port 3000)

|Leptos SSR Shell + UI Components (WASM)
|`appProcess` — WASM bundle served from `/pkg/` within the same binary

|Server Functions + Domain + Persistence
|`appProcess` — server-side execution within the same binary

|Metadata Fetcher
|`appProcess` — outbound HTTPS via `reqwest`

|SQLite Database
|`sqliteFile` on `procastimarks_data` named volume
|===

=== Infrastructure Level 2 — Docker Compose Configuration

The `docker-compose.yml` at the repository root defines the deployment unit.

[source,yaml]
----
# Illustrative — actual values TBD at implementation time
services:
  app:
    image: procastimarks:latest
    restart: unless-stopped
    ports:
      - "127.0.0.1:3000:3000"   # only reachable via reverse proxy
    environment:
      - DATABASE_URL=/data/bookmarks.db
      - API_KEY=${API_KEY}       # set in .env or shell environment
    volumes:
      - procastimarks_data:/data

volumes:
  procastimarks_data:
----

Environment Variables::

[cols="1,1,2", options="header"]
|===
|Variable |Required |Description

|`DATABASE_URL`
|Yes
|Absolute path to the SQLite file inside the container.
Example: `/data/bookmarks.db`.

|`API_KEY`
|Yes
|The secret key embedded in the bookmarklet URL.
Must be a long, random string.  No default value; the application refuses to start
if this variable is unset or empty.
|===

Multi-Stage Docker Build::

[source,dockerfile]
----
# Stage 1 — build
FROM rust:latest AS builder
WORKDIR /app
COPY . .
RUN cargo build --release

# Stage 2 — runtime
FROM debian:bookworm-slim
COPY --from=builder /app/target/release/procastimarks /usr/local/bin/
COPY --from=builder /app/target/site /usr/local/share/procastimarks/site/
EXPOSE 3000
CMD ["procastimarks"]
----

NOTE: The exact base image and WASM asset paths are determined at implementation
time.  The multi-stage principle (build in full Rust image, run in minimal image)
is a fixed architectural decision.
